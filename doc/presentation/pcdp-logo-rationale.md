# PCDP Logo

## Options

Here are a few logo concepts — each captures a different aspect of PCDP:Six directions, each with a different personality:

**A** — Hexagon with the P letterform and a flowing arrow to a code bracket. Suggests transformation. Bold, technical.

**B** — Document → binary, the core PCDP idea in one visual: human-readable spec on the left, machine output on the right, arrow through the middle. Very literal but clear.

**C** — Typographic ring mark with PC/DP stacked. Abstract, scalable to any size, works as a favicon or watermark. The split opacity between PC and DP hints at the paradigm shift.

**D** — Pen nib inside brackets. The nib = authoring/specification; the brackets = code context. A bit more editorial, suits a whitepaper or academic paper.

**E** — Minimal blue square with a bold P and superscript `cd paradigm`. Clean, modern, scales well as an icon. The most "product logo" feel.

**F** — Three-node flow: S (spec) → unlabelled middle node (AI translation) → `</>` (code). The simplest visual expression of the paradigm.

Which direction resonates? I can refine any of these — different colours, proportions, just the mark without wordmark, dark-mode variant, or a completely different idea.

## Chosen: C

The mark refined and shown at three scales — full, small, and icon.

The nib sits inside the brackets: the act of writing (specification) framed by the context of code. The ink drop at the tip is the moment the spec becomes something real. The centre split line is a classic nib detail that also suggests a division — intent on one side, implementation on the other.

A warm terracotta/burnt orange — distinctive from the blue that most developer tooling uses, serious without being corporate, works well on both light and dark backgrounds.
